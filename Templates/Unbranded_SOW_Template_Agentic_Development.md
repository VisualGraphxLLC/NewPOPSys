## 12.4 Prompting Standards

All agent prompts should include:

• Objective (what success looks like)

• Constraints (scope, tech stack, time/budget limits)

• Inputs (links to requirements, API specs, designs)

• Output format (PR, doc update, test report, etc.)

• Done criteria (tests pass, documentation updated, etc.)

## 12.5 Harness / Orchestration

Recommended autonomous loop:

1\) Plan: interpret requirement → break into tasks → estimate risk

2\) Implement: code changes with tests

3\) Verify: run automated checks and produce evidence

4\) Review: peer/agent review → fix issues

5\) Merge: only after gates pass and human approval (if required)

*\[Define tooling (issue tracker, repo workflow, CI), and any required human approval points.\]*

## 12.6 Documentation and Traceability Requirements

Minimum traceability artifacts:

• Ticket/requirement ID referenced in commit messages and PR descriptions

• Updated user-facing documentation for workflow changes

• Updated API documentation for contract changes

• Change log for scope modifications

## 12.7 Security and Data Handling for AI

• Never include secrets (API keys, credentials) in prompts or logs.

• Treat production data as sensitive; use sanitized test data where possible.

• Enforce least-privilege access for any agent tooling.
