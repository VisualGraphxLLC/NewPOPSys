## 1. Company & Product FAQs

### Q: What is PopSystem (NewPOPSys)?

**A**: PopSystem is a payment orchestration platform for K-12 schools that unifies fragmented payment systems into a single, streamlined experience for parents and administrators.

Think of us as "Stripe for school payments" - we sit as an orchestration layer between schools, families, and existing payment service providers (PSPs). Parents get one login, one unified balance, and one place to pay for everything from lunch to athletics to field trips. Schools get centralized administration, real-time reporting, and automated reconciliation across all their payment programs.

**Key Differentiator**: We don't replace existing payment providers - we unify them. Schools can keep their existing PSP relationships while delivering a modern, consolidated experience to parents.

---

### Q: What problem does PopSystem solve?

**A**: We solve the "payment fragmentation crisis" affecting K-12 schools:

**The Problem**:
- Schools use 5-10 different payment systems (lunch, fees, athletics, activities, fundraising, etc.)
- Parents juggle an average of 4.2 different payment logins and accounts
- School administrators spend 15-20 hours per week manually reconciling payments across systems
- $2.8B in fees go uncollected annually due to parent confusion and friction
- Each department selected their own payment solution in isolation, creating operational chaos

**Our Solution**:
- **For Parents**: Single login, unified balance across all programs, one payment for everything
- **For Schools**: Centralized control panel, real-time reporting, automated reconciliation, multi-facility support
- **For Payment Providers**: Maintain existing relationships while improving user experience

**Impact**: Schools reduce admin time by 80%, increase fee collection by 15-25%, and dramatically improve parent satisfaction.

## 2. Business Model FAQs

### Q: How do you make money?

**A**: We operate a dual revenue model combining predictable SaaS subscriptions with optional transaction processing fees.

**Revenue Stream #1: SaaS Subscription (60% of revenue at scale)**

Tiered pricing based on school size and features:

| Tier | Monthly Base | Per Student/Month | Target Segment |
|------|--------------|-------------------|----------------|
| Starter | $99 | $0.50 | Small schools (0-500 students) |
| Professional | $299 | $1.00 | Medium schools, multi-facility |
| Enterprise | Custom | $1.50 | Large districts (2,000+ students) |

**Typical Annual Contract Values**:
- Small school (250 students): $2,700/year
- Medium school (1,000 students): $15,600/year
- Large district (5,000 students): $102,000/year

**Revenue Stream #2: Transaction Processing (40% of revenue at scale)**

Optional payment processing service (schools can use existing PSP or PopSystem):
- Credit/Debit cards: 2.75% + $0.30 per transaction
- ACH/eCheck: 1.0% + $0.25 per transaction
- Average take rate: 2.4% of payment volume

**Blended Revenue per Student**: $22-$24 per year (at scale)
- SaaS: $18/student/year
- Transactions: $4-6/student/year (for participating schools)

**Why This Model Works**:
- SaaS provides predictable, high-margin recurring revenue (85% gross margin)
- Transaction fees provide upside without requiring schools to switch PSPs
- Dual revenue streams reduce dependency on any single model
- Blended gross margin: 75% at scale

## 3. Technology FAQs

### Q: What is your tech stack?

**A**: We've built a modern, cloud-native platform designed for real-time orchestration, scalability, and reliability.

**Architecture Overview**: Three-layer system
1. **Parent/Admin Interface Layer**: Web + mobile apps
2. **Orchestration Engine**: Core payment routing and synchronization
3. **PSP Integration Layer**: Connectors to payment service providers

**Core Technology Stack**:

**Backend**:
- **Language**: Node.js (TypeScript) for orchestration engine
- **Framework**: NestJS for API services
- **Database**: PostgreSQL for transactional data, Redis for caching/real-time sync
- **Message Queue**: RabbitMQ for asynchronous processing
- **API**: RESTful + GraphQL for flexible client access

**Frontend**:
- **Web**: React with Next.js for SSR/SSG
- **Mobile**: React Native for iOS and Android (single codebase)
- **State Management**: Redux for complex state handling
- **UI Framework**: Material-UI with custom theming

**Infrastructure**:
- **Hosting**: AWS (multi-region for redundancy)
- **Containerization**: Docker + Kubernetes for orchestration
- **CI/CD**: GitHub Actions for automated deployment
- **Monitoring**: DataDog for APM, Sentry for error tracking
- **Security**: AWS WAF, encryption at rest and in transit, SOC 2 Type II compliant

**Integration Layer**:
- **PSP Connectors**: Custom-built adapters for each PSP API
- **Sync Engine**: Real-time webhook processing + scheduled batch sync
- **Reconciliation**: Automated matching and conflict resolution
- **Rate Limiting**: Intelligent throttling to respect PSP API limits

**Key Technical Features**:
- Real-time balance aggregation across multiple PSPs
- Intelligent payment routing based on rules and optimization
- Automatic retry and error handling
- Multi-tenant architecture with facility-level isolation
- Comprehensive audit logging for compliance

**Why This Stack**:
- **Scalability**: Handles 10,000+ schools processing millions in transactions
- **Real-time**: Sub-second sync across multiple PSPs
- **Reliability**: 99.9% uptime SLA
- **Developer Velocity**: Modern stack attracts top talent, enables fast iteration
- **Security**: Enterprise-grade security for sensitive payment data
